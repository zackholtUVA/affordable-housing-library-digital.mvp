# Acceptance Checklist

- [ ] App contains all planned routes (`/`, `/explore`, `/options/[slug]`, `/compare`, `/next-steps`, `/basics`)
- [ ] Homepage clearly communicates purpose and entry points
- [ ] Users can browse, filter, and search placeholder options
- [ ] Users can open option detail pages by slug
- [ ] Users can add/remove up to 3 options in compare flow
- [ ] Users can reach action-oriented next-step guidance
- [ ] Basics/glossary support exists with searchable/expandable terms
- [ ] Placeholder-only policy is preserved across content-rich sections
- [ ] Out-of-scope items (auth, backend, calculators, APIs, CMS) are not implemented
- [ ] Dark mode default and light mode toggle both work
- [ ] UI is responsive and keyboard accessible
